Formula Editor: This allows users to define a formula that maps the response variable to the predictor variable(s) for analysis.

Data Grouping and Splitting: Users can dynamically separate data into groups or apply filters, which is reflected in the UI: Split by Group: Enables users to subset data based on one or more column(s).

Test family: In case a linear model is defined you can choose between two test families, parametric or non parametric. The parametric tests require several assumptions such as normally distributed residuals and similar variances across the different groups (homoscedasticity). In case these assumptions are not met, non-parametric or permutation-based tests should be preferred as they rely on fewer distributional assumptions.

Two groups:

    • Parametric (T-Test)

        • Confidence Level: A slider to set the confidence level for the interval, ranging from 0 to 1 (default is 0.95).

        Alternative Hypothesis: A dropdown menu for selecting the type of alternative hypothesis (two-sided, less, or greater).

        Variance Equality: Decide if the variances of the two groups are equal or not. In case they are not equal a Welch test is performed, otherwise a classical Student's t-test.

    • Non-Parametric (Wilcox-Rank-Sum-Test)

        • Confidence Level: A slider to set the confidence level for the interval, ranging from 0 to 1 (default is 0.95).

        Alternative Hypothesis: A dropdown menu for selecting the type of alternative hypothesis (two-sided, less, or greater).

        This test compares the distributions of two groups using ranks instead of raw values and is robust against non-normality and outliers.

Multiple Group Tests:

    • Parametric tests

        • ANOVA: Conduct an analysis of variance test. Supports factorial designs and interaction terms.

        Welch ANOVA: Conduct an analysis of variance test without assuming equal variances across groups. The caveat is that only one predictor can be used.

    • Non-Parametric tests

        • Kruskal-Wallis Test: Non-parametric test for comparing more than two groups. It is based on ranks and does not assume normality. The caveat is that only one predictor can be used.

        Permutation ANOVA (permuco::aovperm):

          A permutation-based alternative to classical ANOVA that evaluates the significance of effects by comparing the observed test statistic to a distribution obtained through random permutations of the data.

          Key characteristics:

          • Does not rely on normality assumptions.
          • Can handle complex designs including interactions (similar to classical ANOVA).
          • Provides p-values based on permutation distributions.
          • More robust against heteroscedasticity and non-normal residuals.

          Important parameters:

          • Number of Permutations: Higher values increase accuracy but also computation time.

          Notes:

          • Results may vary slightly between runs due to random permutations.
          • For reproducibility, a random seed should be set before running the analysis.
          • Suitable as a general replacement when ANOVA assumptions are questionable.

Multiple Comparisons:

  • PostHoc tests

    • PostHoc Tests can be used to investigate which groups differ significantly from each other.

    • Parametric PostHoc tests

      • Tukey HSD (Honestly Significant Difference): A test used to compare all pairs of group means to determine which pairs are significantly different. This test is suitable for balanced designs with equal sample sizes in each group and is often used after ANOVA to perform pairwise comparisons.
      • Least Significant Difference (LSD) test: A post hoc test that compares all pairs of means, but unlike Tukey’s HSD, it does not adjust for multiple comparisons. It is more sensitive but less conservative, which increases the chance of Type I errors (false positives).
      • Scheffe post hoc test: A very conservative test that can be used to make all pairwise comparisons and contrasts between groups. It is robust to unequal sample sizes and can be used after ANOVA with more than two levels, especially when there are complex contrasts.
      • REGW (Revised Generalized Welch’s) post hoc test: This is a robust method for comparing group means when the assumption of equal variances is violated (heteroscedasticity). It is designed to work with unequal sample sizes and unequal variances.
        • Users can also adjust the p-value threshold and choose between balanced or unbalanced designs.

    • Non-Parametric PostHoc tests

      • Kruskal Wallis post hoc test: A non-parametric method used to compare more than two groups. If the Kruskal-Wallis test finds a significant difference, post hoc pairwise comparisons can be performed using the Dunn test or other similar methods to identify which groups differ.
    • Adjusted p-Value Methods:

      • When applicable, users can select the method for adjusting p-values for multiple comparisons, including:

      • Holm: A step-down method for controlling the family-wise error rate (FWER). It adjusts p-values sequentially from the smallest to the largest.
      • Hommel: A more powerful adjustment for controlling the FWER, often used when performing multiple comparisons.
      • Hochberg: A step-up method for controlling the FWER. It is more powerful than Bonferroni but may be less robust under certain conditions.
      • Bonferroni: A conservative method for controlling the FWER. It divides the significance level (alpha) by the number of tests performed to reduce the likelihood of Type I errors.
      • BH (Benjamini-Hochberg): A method to control the false discovery rate (FDR). It is less conservative than Bonferroni and is useful when performing a large number of comparisons.
      • BY (Benjamini-Yekutieli): An extension of the Benjamini-Hochberg method that controls FDR in situations where the tests are dependent.
      • FDR (False Discovery Rate): An alternative to controlling FWER, this method allows for a specified proportion of false positives among the rejected hypotheses.

Pairwise Comparisons: Pairwise comparisons can be used after a global group comparison to determine which specific groups differ from each other. Depending on the selected test family, OpenStats performs either pairwise t-tests (parametric) or pairwise Wilcoxon rank-sum tests (non-parametric). In both cases, paired tests are currently not supported.

  • Parametric: Pairwise t-tests
    Performs pairwise comparisons between group means using t-tests. This option is appropriate when the assumptions of parametric testing are reasonably met, such as approximately normally distributed residuals.
  • Non-Parametric: Pairwise Wilcoxon rank-sum tests
    Performs pairwise comparisons using rank-based Wilcoxon tests. This option is useful when parametric assumptions are not met or when the data contain strong deviations from normality.

Adjusted p method: Select the method used to adjust p-values for multiple testing. This helps control the increased risk of false positives that arises when several pairwise comparisons are performed.

Alternative hypothesis: Defines the direction of the statistical test. Available options are typically two-sided, less, or greater. In most applications, a two-sided test is used unless there is a clear directional research hypothesis.

P-value: Sets the significance threshold used to decide whether a comparison is considered statistically significant. The default value is 0.05.

Run pairwise comparisons: Starts the selected pairwise comparison procedure with the chosen settings. The resulting output reports the pairwise group comparisons together with the corresponding adjusted p-values.
