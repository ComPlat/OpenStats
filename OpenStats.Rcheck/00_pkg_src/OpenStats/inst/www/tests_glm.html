Formula Editor: This allows users to define a formula that maps the response variable to the predictor variable(s) for analysis.

Data Grouping and Splitting: Users can dynamically separate data into groups or apply filters, which is reflected in the UI: Split by Group: Enables users to subset data based on one or more column(s).

Test family: In case a linear model is defined you can choose between two test families, parametric or non parametric. The parametric tests require several assumptions such as normally distributed residuals and similar variances across the different groups (homoscedasticity). In case these assumptions are not met, non-parametric or permutation-based tests should be preferred as they rely on fewer distributional assumptions.

Generalized Linear Model (GLM): Generalized linear models extend classical linear models to response variables that do not follow a normal distribution. They are especially useful for count data, binary outcomes, proportions, and other non-Gaussian response variables. A GLM combines a probability distribution for the response with a link function that relates the expected value of the response to the linear predictor.

Typical examples include:

  • Gaussian: for approximately normally distributed responses.
  • Binomial: for binary outcomes or proportions.
  • Poisson: for count data.

In OpenStats, GLMs can be evaluated using an analysis of deviance and, where appropriate, followed by pairwise comparisons based on estimated marginal means.

Analysis of Deviance (GLM): The analysis of deviance tests whether individual model terms significantly improve the fit of the generalized linear model. It is the GLM analogue of ANOVA for classical linear models.

Instead of comparing sums of squares, this procedure compares changes in deviance. Depending on the model and settings, the test is typically based on likelihood-ratio tests or deviance tests.

Use case: This test is especially useful when more than two groups or multiple predictors are present and you want to assess whether predictors or interactions contribute significantly to the model.

Run Analysis of Deviance: Starts the analysis of deviance for the fitted generalized linear model and reports the significance of the model terms.

Pairwise Comparisons for GLMs: After fitting a generalized linear model, pairwise comparisons can be performed to investigate which groups differ from each other. These comparisons are typically based on estimated marginal means and account for the fitted GLM structure.

This approach is particularly useful when the global model indicates significant group differences and a more detailed comparison between levels of a factor is required.

Choose an adjustment method: Select the method used to adjust p-values for multiple comparisons. This is important because running several pairwise tests increases the risk of false positives.

  • Tukey: Common choice for all pairwise group comparisons.
  • Sidak: Multiple comparison correction similar to Bonferroni, but slightly less conservative.
  • Bonferroni: Conservative correction controlling the family-wise error rate.
  • Scheffé: Very conservative method, suitable for more general contrasts.
  • None: No p-value adjustment is applied.
  • FDR: Controls the false discovery rate.
  • Holm: Step-down procedure controlling the family-wise error rate.
  • Hochberg: Step-up procedure controlling the family-wise error rate.
  • Hommel: A more advanced method for controlling the family-wise error rate.

Run PostHoc test: Starts the pairwise comparison procedure for the fitted generalized linear model using the selected p-value adjustment method.
