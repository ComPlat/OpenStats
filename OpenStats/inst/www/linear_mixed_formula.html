1. Overview

A Linear Mixed Model (LMM) extends the ordinary linear model by allowing some coefficients to vary across groups. This is needed whenever observations are not independent — for example when the same subject is measured repeatedly, or when samples come from different batches, schools, or sites.

The formula has two parts on the right-hand side:

  • Fixed effects — the predictors you care about (same as in a regular linear model).
  • Random effects — written inside ( | ), they account for the grouping structure.

2. Getting Started

  1. Select the response variable from the left-side dropdown (e.g., blood_pressure).
  2. Click predictor buttons to build the fixed-effects part (e.g., treatment + time).
  3. Add a random-effect term using the | button: type or drop (1|patient).
  4. Click Create Statistical Model.

3. The | Operator — Random Effects

The vertical bar | separates the what varies from the grouping variable:

(what_varies | grouping_variable)

SyntaxMeaningWhen to use
(1 | group) Random intercept — each group gets its own baseline Subjects/sites differ in their average level
(predictor | group) Random slope — each group gets its own slope for that predictor Subjects respond differently to the predictor
(1 + predictor | group) Random intercept and random slope Groups differ both in baseline and in response

Rule of thumb: always start with a random intercept (1|group). Add a random slope only if you have enough data per group and a scientific reason to expect that the effect of the predictor differs across groups.

Example 1 — Random Intercept: blood_pressure ~ time + (1|patient)

Each patient has a different baseline blood pressure. The fixed effect of time estimates the average change per time unit across all patients. (1|patient) lets the model shift up or down for each patient without treating those shifts as error.

Interpretation
  • time: Blood pressure decreases by ~4.2 units per time step on average (p = 0.002).
  • patient variance (11.2 SD): Patients differ quite a lot in their baseline — much more than the residual variation (2.4 SD) within a patient. This confirms that accounting for the patient grouping was necessary.
  • Degrees of freedom: LMMs use Satterthwaite (or Kenward-Roger) approximations for df, so they are not whole numbers. This is expected and correct.

Example 2 — Fixed Interaction + Random Intercept: blood_pressure ~ treatment * time + (1|patient)

Tests whether the slope of blood pressure over time differs between treatment groups. The interaction term treatment:time captures exactly this difference in slopes.

  • treatmentdrug:time: The drug group drops an additional ~7.7 units per time step compared to placebo (total slope ≈ −11.2). The interaction is significant (p = 0.008), meaning the drug works differently over time than placebo.
  • treatmentdrug (baseline): The drug group starts ~19.5 units higher at time 0, but this is not significant — the groups differ mainly in how they change, not in their starting point.

4. Common Mistakes

  • Forgetting parentheses: 1|patient without ( ) is not valid syntax. Always write (1|patient).
  • Using the wrong grouping variable: The variable inside ( | ) must be the column that defines the groups (e.g., subject ID, batch ID, site), not a predictor of interest.
  • Too few groups: Random effects need enough groups to estimate variance. As a rough rule, aim for at least 5–6 levels in the grouping variable. With fewer groups, consider treating the grouping variable as a fixed effect instead.
  • Convergence warnings: If the model warns about convergence, try simplifying the random-effects structure — remove random slopes before removing random intercepts.
